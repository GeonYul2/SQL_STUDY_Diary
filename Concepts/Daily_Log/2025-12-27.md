## 👥 1. 인플루언서 마케팅 후보 찾기 (Level 5)
**목표**: 친구의 친구(2-hop) 수가 많은 상위 유저 5명 추출.

### ❌ 초기 시도 (실패)
`user_a_id` -> `user_b_id` 단방향만 고려함. 친구 관계는 양방향(A가 B의 친구면 B도 A의 친구)임을 간과.

### ✅ 해결책 (Union All로 무방향화)
먼저 모든 관계를 `(user_id, friend_id)` 형태로 풀어서 2배로 불린 뒤 시작한다.

```sql
WITH friends AS (
    -- 1. 관계 펼치기 (A->B, B->A 모두 포함)
    SELECT user_a_id AS user_id, user_b_id AS friend_id FROM edges
    UNION ALL
    SELECT user_b_id AS user_id, user_a_id AS friend_id FROM edges
),
degree AS (
    -- 2. 내 직접 친구 수
    SELECT user_id, COUNT(*) AS friends
    FROM friends
    GROUP BY user_id
),
fof_sum AS (
    -- 3. 내 친구들의 친구 수 합산
    SELECT f.user_id, SUM(d.friends) AS friends_of_friends
    FROM friends f
    JOIN degree d ON f.friend_id = d.user_id
    GROUP BY f.user_id
)
SELECT 
    d.user_id, d.friends, f.friends_of_friends,
    ROUND(f.friends_of_friends / d.friends, 2) AS ratio
FROM degree d
JOIN fof_sum f ON d.user_id = f.user_id
WHERE d.friends >= 100
ORDER BY ratio DESC
LIMIT 5;
```

## 📦 2. 연도별 배송 업체 이용 내역

### 💡 Core Insight: Pivot Pattern
`Standard`, `Express` 등의 행(Row) 값을 컬럼(Column)으로 변환하여 합산해야 함.

### Recommended Code
`SUM(CASE WHEN ... THEN 1 ELSE 0 END)`을 사용하면 CTE 없이 한 번에 해결 가능.

```sql
SELECT 
    YEAR(purchased_at) AS year,
    -- Standard는 반품 건수도 포함
    SUM(CASE WHEN shipping_method = 'Standard' THEN 1 ELSE 0 END) + SUM(is_returned) AS standard,
    SUM(CASE WHEN shipping_method = 'Express' THEN 1 ELSE 0 END) AS express,
    SUM(CASE WHEN shipping_method = 'Overnight' THEN 1 ELSE 0 END) AS overnight
FROM transactions
WHERE shipping_method IS NOT NULL
GROUP BY YEAR(purchased_at)
ORDER BY year;
```
