## 😷 1. 이틀 연속 미세먼지가 나빠진 날
**문제**: 오늘(m1), 내일(m2), 모레(m3)의 미세먼지 농도가 점점 증가하는 날짜 찾기.

### 🔑 Key Point: 3-way Self Join
`DATE_ADD`를 사용하여 연속된 날짜를 조인 조건으로 삼는다.

```sql
SELECT m3.measured_at as date_alert
FROM measurements m1
JOIN measurements m2 ON DATE_ADD(m1.measured_at, INTERVAL 1 DAY) = m2.measured_at 
JOIN measurements m3 ON DATE_ADD(m2.measured_at, INTERVAL 1 DAY) = m3.measured_at
WHERE m3.pm10 >= 30 
  AND m1.pm10 < m2.pm10 
  AND m2.pm10 < m3.pm10
ORDER BY date_alert;
```

## 🏐 2. 배구 선수 분석

### 2.1 두 대회 연속 출전 선수
4년 주기 올림픽(`year + 4`)을 이용한 Self Join.

```sql
SELECT DISTINCT v1.athlete_id AS id, v1.name
FROM vb v1
JOIN vb v2 
    ON v1.athlete_id = v2.athlete_id 
    AND v1.year + 4 = v2.year
ORDER BY id;
```

### 2.2 메달 목록 합치기 (GROUP_CONCAT)
여러 행으로 된 메달 정보를 `,`로 구분된 하나의 문자열로 합침.

```sql
SELECT id, name,
       GROUP_CONCAT(medal ORDER BY medal SEPARATOR ',') AS medals
FROM vb
GROUP BY id, name;
```

## 🎮 3. 장르/연도별 게임 점수 (Pivot)

### 🚨 Error Case
`AVG` 함수를 `CASE WHEN` 밖에서 사용할 때 `GROUP BY` 문제 발생.

### ✅ Correct Pattern
`AVG` 안에 `CASE WHEN`을 넣어서 집계해야 한다. "연도별 점수"는 컬럼으로 펼쳐지므로 결과에는 `Year` 행이 남지 않는다.

```sql
SELECT
  ge.name AS genre,
  ROUND(AVG(CASE WHEN ga.year = 2011 THEN ga.critic_score END), 2) AS score_2011,
  ROUND(AVG(CASE WHEN ga.year = 2012 THEN ga.critic_score END), 2) AS score_2012
FROM games AS ga
JOIN genres AS ge ON ga.genre_id = ge.genre_id
GROUP BY ge.name;
```
> **Rule**: "그룹 안에서 조건별 값을 컬럼으로 만들고 싶다면 -> **집계함수(CASE WHEN...)**"
